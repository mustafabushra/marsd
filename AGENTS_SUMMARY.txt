EXECUTION MODEL:
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

Every task follows:

1. CREATE TASK
   └─ Add security checklist items (required)

2. ASSIGN TO AGENT
   └─ Validate checklist exists
   └─ Route to matching agent

3. PRE-EXECUTION
   └─ Block if ANY critical item unverified ⛔

4. EXECUTE
   └─ Agent implements task
   └─ Marks security items verified with evidence

5. POST-EXECUTION
   └─ Verify all items completed

6. AUDIT LOG
   └─ Immutable record of execution
