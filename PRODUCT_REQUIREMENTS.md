# متطلبات المنتج - منصة مرصد SaaS

> تاريخ التحديث: 2026-07-23

## مقدمة

هذه الوثيقة تحتوي على جميع المتطلبات المنطقية والعملية لمنصة مرصد. تم تصميمها لتكون المرجع النهائي للمطورين والمتخذي القرار.

---

# 1) لوحة التحكم (Dashboard)

## الهدف
إعطاء الشركة نظرة سريعة عن وضعها الحالي.

## من يستطيع الوصول؟
- Owner
- Admin
- Manager
- Viewer (حسب الصلاحية)

## متى تظهر؟
بعد تسجيل الدخول فقط.

## ماذا تعرض؟
- Trust Score
- عدد Credits
- الباقة الحالية
- عدد التقارير المرسلة
- عدد التقارير المستلمة
- الشركات في قائمة المراقبة
- آخر الإشعارات

## الأزرار

### إضافة تقرير

**يظهر إذا:**
- الاشتراك نشط
- الشركة مفعلة
- المستخدم لديه صلاحية
- Credits > 0

**يختفي إذا:**
- الحساب معلق
- الاشتراك منتهي
- المستخدم Viewer
- Credits = 0

---

# 2) البحث عن الشركات

## الهدف
البحث عن أي شركة داخل مرصد.

## مصدر البيانات
- الشركات المسجلة
- الشركات المضافة عبر التقارير
- الشركات التي أضافتها إدارة مرصد

## البحث بـ
- الاسم
- السجل التجاري
- المدينة
- القطاع

## زر عرض الشركة

**يظهر إذا:**
- الشركة موجودة

## زر إرسال تقرير

**يظهر إذا:**
- ليست شركتك
- لديك Credits
- الاشتراك يسمح

**يختفي إذا:**
- الشركة هي شركتك
- Credits = 0
- الاشتراك منتهي

---

# 3) إضافة تقرير

## الهدف
إرسال تقييم عن شركة أخرى.

## الشروط - يمنع الإرسال إذا:
- لا يوجد Credits
- الاشتراك منتهي
- التقرير مكرر خلال 90 يوم (BR-05)
- الشركة غير مفعلة
- المستخدم Viewer
- الشركة المستهدفة هي نفس شركة المستخدم

## عند النجاح
- خصم Credit واحد
- إنشاء Notification
- إرسال التقرير للمراجعة
- تغيير الحالة إلى Pending
- تحديث last_report_date

## عند الرفض (Rejected)
- إرجاع Credit
- إنشاء Notification برسالة الرفض

---

# 4) تقاريري المرسلة

## تعرض
- Pending
- Approved
- Rejected

## المسموح
- عرض التقرير
- عرض سبب الرفض (إذا Rejected)

## يمنع
- تعديل التقرير بعد الإرسال

## إذا Rejected

**يسمح:**
- إرسال نسخة جديدة

---

# 5) قائمة المراقبة (Watchlist)

## إضافة
**يسمح إذا:**
- الشركة ليست موجودة بالقائمة
- الشركة ليست شركتك

## إزالة
**مسموح دائماً**

## عند اعتماد تقرير جديد
النظام يرسل Notification لكل المراقبين.

---

# 6) مقارنة الشركات

## حسب الباقة

| الباقة | الحد |
|-------|------|
| Free | حتى شركتين |
| Basic | حتى 5 شركات |
| Pro | حتى 10 |
| Enterprise | غير محدود |

---

# 7) إدارة المستخدمين

## Owner فقط يستطيع:
- دعوة مستخدم
- حذف مستخدم
- تغيير الصلاحيات

## Admin يستطيع:
- إضافة مستخدم

## Admin لا يستطيع:
- تغيير Owner

## Viewer
- مشاهدة فقط
- لا إضافة أي بيانات
- لا تعديل أي بيانات

---

# 8) الاشتراك

## يعرض
- الباقة الحالية
- تاريخ انتهاء الاشتراك
- عدد Credits المتبقي
- الفواتير السابقة
- جدول المقارنة بين الباقات

## إذا انتهت الباقة

**تعطيل:**
- إضافة تقرير
- مقارنة الشركات (حسب الباقة)
- بعض الميزات المدفوعة

**السماح بـ:**
- عرض البيانات
- تجديد الاشتراك

---

# 9) ملف الشركة (Company Profile)

## يسمح بتعديل
- الشعار
- الهاتف
- البريد الإلكتروني
- الموقع الإلكتروني
- النشاط/الوصف

## لا يسمح بتعديل (إلا عبر الإدارة)
- رقم السجل التجاري
- رقم المنشأة
- Tenant ID
- تاريخ التأسيس

---

# قواعس عامة للنظام (Global Conditions)

## إذا المستخدم غير مسجل
→ تحويل إلى Login

## إذا الحساب معلق
- إظهار رسالة تحذير
- منع كل العمليات
- السماح فقط: عرض البيانات، تحديث الدفع

## إذا الاشتراك منتهي
**السماح فقط:**
- تسجيل الدخول
- عرض البيانات
- تجديد الاشتراك

## إذا Credits = 0
**تعطيل:**
- إضافة تقرير
- البحث المتقدم (إذا كانت الباقة تعتمد على Credits)

## إذا الشركة غير مفعلة
**منع:**
- إضافة تقارير
- دعوة مستخدمين
- إدارة الاشتراك

## إذا المستخدم Viewer
**منع:**
- إضافة تقرير
- إدارة المستخدمين
- تعديل الشركة
- تعديل الاشتراك
- الوصول للفواتير

# حالات التقرير (Report Status Flow)

```
Pending
   ↓
Under Review (by Admin)
   ↓
   ├─ Approved → Published
   └─ Rejected → (User can resubmit)
```

# الإشعارات (Notifications)

## أنواع الإشعارات المهمة

| الحدث | الفئة المستقبلة | الرسالة |
|------|-----------------|---------|
| تم اعتماد التقرير | المُبلِّغ | "تم اعتماد تقريرك عن [الشركة]" |
| تم إضافة تقرير عن شركة في القائمة | المراقبين | "تقرير جديد عن [الشركة] في قائمتك" |
| انتهاء الاشتراك | صاحب الحساب | "انتهى اشتراكك، جدد الآن" |
| Credits منخفضة | صاحب الحساب | "لديك [X] Credits متبقي" |

# الأمان والخصوصية

## عدم الكشف عن الهوية (Anonymization)
- لا يُعاد `reporter_tenant_id` إلا للمراجع والإدارة
- التقارير تُعرض كمجموع، ليس فردي
- اسم المُبلِّغ لا يظهر للشركة المستهدفة

## Row-Level Security (RLS)
- كل tenant يرى فقط بيانات نفسه
- الإدارة ترى كل البيانات
- لا توجد مشاركة بيانات بين الـ tenants
