## القسم التنفيذي المختصر

**المنتج:** منصة SaaS سحابية B2B لتقييم موثوقية الشركات في السوق السعودي.

**النموذج:** اشتراكات شهرية/سنوية (4 باقات: مجاني + أساسي + احترافي + مؤسسات).

**الآلية:** 
- **البيانات الرسمية** (30%): السجل التجاري، الحالة، العمر
- **بيانات المجتمع التجاري** (70%): تقارير من شركات تعاملت فعلياً
- **مؤشر ثقة محسوب**: خوارزمياً من البيانات المعتمدة فقط

**الحماية:** كل تقرير يخضع لمراجعة إدارية إلزامية قبل الاعتماد — هوية المُبلِّغ **محمية تماماً** (إحصائيات مجمّعة فقط).

## 🏗️ البنية المعمارية الأساسية

**طبقة العميل:** Next.js + React (App Router)  
**طبقة الخادم:** NestJS + Prisma  
**قاعدة البيانات:** PostgreSQL + RLS (Row-Level Security)  
**التخزين:** AWS S3 (المستندات/الفواتير)  
**الكاش والطوابير:** Redis + BullMQ  
**البريد:** AWS SES  
**الدفع:** Moyasar أو Tap (يُعتمد أسبوع 1)  
**الحافة:** Cloudflare (WAF + CDN + DDoS)

## 📊 أنماط البيانات الأساسية

### 3 أنواع عزل
1. **Tenant (الشركة المشترِكة):** عزل كامل لكل شركة (RLS + tenant_id)
2. **Company (موضوع التقييم):** قد تكون مشترِكة أم لا
3. **Report (التقرير):** مرتبط بـ Tenant + Company، حالة: pending → review → approved/rejected

### حالات عرض التقرير (إلزامي من التصميم)
| عدد التقارير المعتمدة | الحالة | ما يُعرض |
|---|---|---|
| ≥ 5 | تقييم موثوق | الرقم + الفئة + الإحصائيات كاملة |
| 2–4 | تقييم أولي | فئة استرشادية فقط (بلون تحذير) |
| 0–1 | بيانات غير كافية | رسالة "لا توجد بيانات معتمدة كافية" |
| (أي حالة) + باقة مجانية | مقفل | البيانات الرسمية ظاهرة + المؤشر **مموّه** + دعوة ترقية |

## 🔒 متطلبات الأمان (لا تفاوض)

**OWASP Top 10:**
- SQL Injection: Prisma (parameterized)
- XSS: React.createElement (no innerHTML)
- CSRF: SameSite cookies
- SSRF, XXE, Command Injection, Template Injection, Path Traversal, IDOR

**المصادقة:**
- Argon2id لكلمات المرور
- JWT: 15 دقيقة (Access) + 7 أيام (Refresh) بتدوير
- قفل الحساب بعد 5 محاولات فاشلة

**النقل والتخزين:**
- TLS 1.3 إجباري (HSTS)
- AES-256 لـ RDS/S3

**عزل Multi-Tenant:**
- RLS على كل الجداول الحاملة لـ tenant_id
- لا يمكن لشركة واحدة الوصول لبيانات شركة أخرى حتى بخطأ برمجي

**الخصوصية:**
- هوية المُبلِّغ **محمية معمارياً** — لا توجد طريقة للكشف عنها حتى من Logs
- إحصائيات مجمّعة فقط

**Audit Trail:**
- كل عملية حساسة تُسجل (من، ماذا، متى، IP) — append-only

## 🚨 قواعد العمل الحرجة (Business Rules)

| BR | القاعدة |
|---|---|
| **BR-01** | لا تقرير يُحتسب إلا بعد اعتماده إدارياً |
| **BR-02** | هوية المُبلِّغ **لن تُعرض أبداً** — إحصائيات مجمّعة فقط |
| **BR-03** | أقل من 5 تقارير = "بيانات غير كافية" (قابل للضبط) |
| **BR-04** | سقف مساهمة أي تقرير واحد: 15% من وزن طبقة المجتمع |
| **BR-05** | شركة واحدة: تقرير واحد فقط/90 يوماً لنفس الشركة المستهدفة |
| **BR-06** | 2–4 تقارير = تقييم **أولي** (مع تحذير) بدون رقم نهائي |
| **BR-07** | الباقة المجانية: البيانات الرسمية ظاهرة، المؤشر + بيانات المجتمع **مقفولة** |
| **BR-08** | تقرير مرفوض = عودة للمرسل مع السبب، إعادة إرسال **مرة واحدة** |
| **BR-10** | تجاوز حد الباقة = رسالة ترقية، **لا رسم تلقائي** |
| **BR-11** | كل عملية حساسة في Audit Log (غير قابلة للتعديل) |

## 💡 معادلة مؤشر الثقة (Trust Score v1)

```
TrustScore = clamp(
    W_official × S_official + W_community × S_community,
    0, 100
)

الأوزان الافتراضية:
  W_official   = 0.30
  W_community  = 0.70
```

**S_official:** حالة السجل (50%) + عمر الشركة (30%) + اكتمال البيانات (20%)

**S_community:** لكل تقرير معتمد i:
- score_i = 100 (التزام كامل) | 60 (تأخير ≤30ي) | 35 (31–90ي) | 10 (>90ي) | 0 (تعثر)
- weight_i = min(recency_i × 1.0, cap) حيث recency = 0.5^(عمر بالأشهر/12)
- S_community = Σ(score_i × weight_i) / Σ(weight_i)

**الفئات:**
- 70–100: أخضر (مخاطر منخفضة)
- 40–69: برتقالي (مخاطر متوسطة)
- 0–39: أحمر (مخاطر مرتفعة)
